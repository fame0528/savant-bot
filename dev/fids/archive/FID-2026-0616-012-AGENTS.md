## Lessons Learned

1. **AGENTS.md is the project-specific layer, not a duplicate of ECHO.md.** Upstream Savant-core repeats the protocol inline (its AGENT-PROMPT.md is ~3000 words including the laws). savant-bot/AGENTS.md is the savant-bot-specific overlay (~1100 words). Future agents: read ECHO.md first, then AGENTS.md for project deltas.

2. **The dual-versioning rule belongs in AGENTS.md, not just in CHANGELOG preamble.** A CHANGELOG-only rule is invisible to agents reading the codebase at boot. AGENTS.md's Section 2 makes the rule discoverable at project root.

3. **The production-entry-point file list is project-specific.** ECHO.md Law 4 mandates grep-verification but does not list WHICH files count as production entry points. Savant-core lists 5 files (ignition.rs, swarm.rs, heartbeat.rs, stream.rs, reactor.rs); savant-bot lists 5 different files (main.rs, lib.rs, commands/mod.rs, data.rs, moderation/poller.rs). Listing per-project prevents agents from grep-orphaning new features.

4. **Hybrid reference model beats full rewrite.** AGENTS.md uses 1-line references to ECHO.md (universal universal laws) and full content for savant-bot-specific sections. The alternative — copying all 15 laws into AGENTS.md — wastes 700 lines and risks drift when upstream ECHO.md updates.

5. **Write_file lesson: 2nd occurrence avoided.** This FID updates dev/LEARNINGS.md using `str_replace` with the existing file marker (`<!-- Add new entries above this line -->`). The data-loss lesson from session 2026-06-16-1922 propagated correctly — no recovery was needed.
